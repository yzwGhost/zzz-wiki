## Context

当前卡片组件共用 `catalog-card` 基础样式，但封面区的文本承载方式存在分叉：代理人使用纯图片加正文标题，驱动盘使用海报式封面，音擎已经改为底部渐变信息层。用户要求把代理人和驱动盘也收敛为相同模式，同时保持现有路由、收藏按钮和正文信息结构不变。

## Goals / Non-Goals

**Goals:**
- 统一三类资料卡片的封面信息结构
- 为封面文字提供稳定的渐变信息承载层
- 将改动限制在卡片组件和全局卡片样式

**Non-Goals:**
- 不重构列表页数据流
- 不新增详情页布局改造
- 不引入新的展示组件抽象

## Decisions

- 继续沿用现有 `catalog-card` 基础类，并为不同卡片类型增加轻量修饰类，而不是重新抽象一个通用卡片组件。
  - 这样可以最小化影响范围，避免连带改动列表页与详情页逻辑。
- 将名称从正文区上移到封面底部信息层，正文区仅保留元信息和摘要。
  - 这样可以统一视觉节奏，并消除图片与正文文案的视觉冲突。
- 驱动盘保留海报化背景，但增加与音擎一致的底部信息层。
  - 这样既保留模块辨识度，也能统一信息承载模式。

## Risks / Trade-offs

- [Risk] 代理人封面图局部较亮时，底部信息层对比度不足 → Mitigation: 使用较深的渐变遮罩并保持文字只落在遮罩区
- [Risk] 正文减少一个标题后，信息密度变化导致高度不一致 → Mitigation: 继续依赖统一的正文最小高度和摘要截断规则
